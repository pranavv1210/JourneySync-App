# Security Policy

## Supported Versions

Security fixes are prioritized for the latest commit on the default branch.

## Reporting a Vulnerability

Please report vulnerabilities privately and responsibly.

Contact:

- `pranavv736@gmail.com`

Include:

- Affected component/screen/service
- Reproduction steps or proof of concept
- Estimated severity and potential impact
- Suggested mitigation (if available)

## Response Process

- Initial acknowledgement target: within 72 hours
- Triage and severity assessment: as soon as reproducible
- Fix timeline: based on severity and exploitability
- Disclosure: coordinated after a fix is available

## Scope Highlights

Areas that are especially security-sensitive in JourneySync:

- Authentication and phone verification flow
- Session persistence and token storage
- Supabase RLS/policies and table permissions
- GPS/location data handling and SOS context
- Uploaded media and storage bucket access

## Responsible Disclosure

Please do not disclose vulnerabilities publicly until a fix has been released.
